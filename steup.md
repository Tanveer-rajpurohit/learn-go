# Go Backend вЂ” General Setup Guide

A reusable, step-by-step reference for initializing any Go backend project from scratch.  
Follow each step in order вЂ” nothing is skipped or assumed.

---

## Step 1 вЂ” Initialize the Project

```bash
mkdir <project-name> && cd <project-name>
go mod init github.com/<your-username>/<project-name>
```

---

## Step 2 вЂ” Install Core Dependencies

Install these for every Go backend project:

```bash
go get github.com/go-chi/chi/v5
go get github.com/go-chi/cors
go get github.com/joho/godotenv
```

Install global CLI tools (run once on your machine, not per project):

```bash
go install github.com/air-verse/air@latest
```

Initialize hot reload config:

```bash
air init
```

> `air` watches your files and auto-restarts the server on every save.  
> The generated `.air.toml` config works out of the box for most projects.

## Step 4 вЂ” Boilerplate Code

### `cmd/server/main.go`

```go
package main

import (
    "log"
    "net/http"
    "os"

    "github.com/<your-username>/<project-name>/internal/routes"
    "github.com/go-chi/chi/v5"
    "github.com/go-chi/cors"
    "github.com/joho/godotenv"
)

func main() {
    godotenv.Load()

    PORT := os.Getenv("PORT")
    if PORT == "" {
        PORT = "8080"
    }

    router := chi.NewRouter()
    router.Use(cors.Handler(cors.Options{
        AllowedOrigins:   []string{"*"},
        AllowedMethods:   []string{"GET", "POST", "PUT", "DELETE", "OPTIONS"},
        AllowedHeaders:   []string{"*"},
        ExposedHeaders:   []string{"Link"},
        AllowCredentials: false,
        MaxAge:           300,
    }))

    v1router := chi.NewRouter()
    routes.SetupRouter(v1router)
    router.Mount("/api/v1", v1router)

    srv := &http.Server{
        Handler: router,
        Addr:    ":" + PORT,
    }

    log.Printf("Server starting on port %v", PORT)
    if err := srv.ListenAndServe(); err != nil {
        log.Fatal("Failed to start server: ", err)
    }
}
```

### `internal/routes/routes.go`


### `internal/utils/response.go`

```go
package utils

import (
    "encoding/json"
    "net/http"
)

func ResponseWithJSON(w http.ResponseWriter, statusCode int, data interface{}) {
    response, err := json.Marshal(data)
    if err != nil {
        http.Error(w, "Failed to marshal JSON", http.StatusInternalServerError)
        return
    }
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(statusCode)
    w.Write(response)
}

func ResponseWithError(w http.ResponseWriter, status int, message string) {
    ResponseWithJSON(w, status, struct {
        Error string `json:"error"`
    }{Error: message})
}
```

## Step 5 вЂ” Database Setup

### Install dependencies

```bash
go get github.com/jackc/pgx/v5
go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest
go install -tags 'postgres' github.com/golang-migrate/migrate/v4/cmd/migrate@latest
```

### Create your first migration

```bash
migrate create -ext sql -dir db/migrations create_users
```

This generates two files inside `db/migrations/`:

**`*_create_users.up.sql`** вЂ” applied when running `up`:
```sql
CREATE TABLE users (
    id    SERIAL PRIMARY KEY,
    name  TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL
);
```

**`*_create_users.down.sql`** вЂ” applied when rolling back:
```sql
DROP TABLE users;
```

Run migrations against your database:

```bash
migrate -path db/migrations -database "postgres://user:password@localhost:5432/dbname?sslmode=disable" up
```

## Step 6 вЂ” sqlc Setup (Type-Safe DB Queries)

### `sqlc.yaml` вЂ” place at project root

```yaml
version: "2"
sql:
  - schema: "db/migrations"
    queries: "db/query"
    engine: "postgresql"
    gen:
      go:
        package: "db"
        out: "internal/db"
        sql_package: "pgx/v5"
        emit_json_tags: true
        json_tags_case_style: "snake"
```

### `db/query/users.sql` вЂ” write your queries here

```sql
-- name: CreateUser :one
INSERT INTO users (name, email)
VALUES ($1, $2)
RETURNING id, name, email;

-- name: GetUsers :many
SELECT id, name, email FROM users;

-- name: GetUser :one
SELECT id, name, email FROM users WHERE id = $1;

-- name: UpdateUser :one
UPDATE users
SET name = $2, email = $3
WHERE id = $1
RETURNING id, name, email;

-- name: DeleteUser :exec
DELETE FROM users WHERE id = $1;
```

Generate Go code from your SQL:

```bash
sqlc generate
```

> Output goes into `internal/db/` вЂ” never edit these files manually.  
> Every time you change a query or schema, re-run `sqlc generate`.

## Step 7 вЂ” Run the Server

```bash
# Development (hot reload)
air

# Manual build and run
go build -o app ./cmd/server && ./app
```

Hit `GET /api/v1/health` вЂ” you should get:

```json
{ "status": "ok" }
```

## Quick Reference вЂ” Common Commands

| Task | Command |
|---|---|
| Run dev server | `air` |
| Create migration | `migrate create -ext sql -dir db/migrations <name>` |
| Apply migrations | `migrate -path db/migrations -database "$DATABASE_URL" up` |
| Rollback migration | `migrate -path db/migrations -database "$DATABASE_URL" down 1` |
| Generate DB code | `sqlc generate` |
| Add a dependency | `go get github.com/<package>` |
| Tidy dependencies | `go mod tidy` |
